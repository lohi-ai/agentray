AgentRay · Agent Teams

How do Agent Teams work?

Group your existing agents into a team, pick one lead, hand out work as cards on a kanban board — then just talk to the lead. The lead orchestrates the members and keeps the board up to date.

Four steps, one working loop

No code and no special configuration — a team is just a way to organize the agents you already have.

1

Form the team

Go to Teams, create a team, and add at least 2 project agents to the roster.

2

Pick a lead

Click Make lead on one member. Only the lead is granted the orchestration skill and the board tool.

3

Add work cards

Each piece of work is a card on the kanban board, optionally assigned to a specific member.

4

Chat with the lead

Ask the lead to “work the board”. The lead delegates to each member, collects results, and moves cards to the next column.

What is the lead — and why only one?

In every team, exactly one agent holds the lead role. The platform injects the orchestrator skill and the team_board tool into the lead only — the other members can't see the board and can't delegate work on their own.

Roster — example

The lead orchestrates

It reads the board, picks cards, delegates each one to the best-suited member, collects results, and updates card status.

Members only do assigned work

Each assignment runs as its own delegate run under that agent's own permissions, tools, and persona.

One board, two hands

You and the lead work the same board: the lead moves cards through the team_board tool, you drag cards directly in the UI — same underlying data.

Every run is recorded

Each delegate run shows up in the Agent monitor with tokens, cost, and a full log — you can see who did what, and when.

The safety rail: delegated ≠ allowed to publish

Runs the lead delegates (delegate runs) count as unattended runs — same as scheduled runs. At the Suggest or Scheduled autonomy rungs (the default), the platform automatically strips the Fetch / HTTP publishing tools from these runs: members stop at drafts and can never publish externally on their own.

Only when the workspace deliberately opts into the Auto rung (under Set up → Permissions → Autonomy) do unattended runs keep their publishing tools — and the agent then owes an audit trail of everything it published.

Get started in 2 minutes

  1. Open Teams in the navigation and click New team. Teams belong to the current project.
  2. Add at least 2 agents to the roster. Use existing agents or install from the Marketplace (Data Analyst, Marketing Lead…).
  3. Click Make lead on the agent you want in charge.
  4. Add a few work cards with New card — one concrete task per card, assign if you like.
  5. Open Chat, pick the lead, and ask: “Work the board for team X, please.” Watch the runs in Agent monitor.
